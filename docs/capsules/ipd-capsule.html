What only patient-level data can tell you

Aggregate meta-analysis can pool the average effect, but it answers questions about who benefits using trial averages — and trial averages lie. The slope of effect against a patient covariate across trials can point the opposite way to the slope within each trial. IPD keeps the two apart; this capsule shows the gap.

Two-stage IPD — living · live re-run illustrative demo data

What this means — in plain language

With each patient's data you can ask who benefits, not just the average effect. This shows how the effect of a patient factor inside trials can point the opposite way to the pattern across trials — a trap only patient-level data avoids.

01

The ecological trap

within vs across

Two slopes, one covariate

EffectEach trial's treatment effect (mean difference), with its standard error. Within-trial slopeHow the effect changes with a patient covariate inside a trial — estimable only from IPD. Pooled, it is the unbiased patient-level effect modification. Across-trial slopeHow trials' effects change with their mean covariate — all that aggregate data can offer. It mixes effect modification with trial-level confounding. Aggregation biasWhen the two slopes disagree, the across-trial (ecological) answer is misleading; a one-stage IPD model includes both terms to keep them separate.

A two-stage IPD analysis estimates each effect/slope within a trial, then pools (shown here). A one-stage model fits all patients at once with trial as a stratum; for the main effect the two agree closely, but the one-stage model is preferred for sparse/binary data and for cleanly separating within- from across-trial covariate effects.

05

Aggregation (ecological) bias

—

The difference between the across-trial and pooled within-trial slope is the aggregation bias. A one-stage IPD model that includes only a single (aggregate) covariate term inherits this bias; the fix is to include the patient-centred covariate and the trial-mean covariate as separate terms.

Counting clusters, not people

When you randomise whole clinics or villages, patients in the same cluster resemble each other — so each one carries less independent information than a lone participant would. Analyse a cluster trial as if everyone were independent and the confidence interval comes out too narrow; the design effect is the correction, and it can turn a "significant" result back into an open question.

What this means — in plain language

When whole clinics or villages are randomised, people in the same group resemble each other, so each one carries less information. This shows how much that shrinks the real evidence — enough, sometimes, to turn a 'significant' result back into an open question.

Why a person isn't worth a person

ICC & DEFF

The design effect

ICC (ρ)Intracluster correlation: how alike outcomes are within a cluster. Even 0.01–0.05 matters when clusters are large. DEFFDesign effect = 1 + (m̄ − 1)·ρ, where m̄ is the average cluster size. The variance is inflated by this factor. Effective NN_eff = N / DEFF — the number of independent participants the trial is really worth. Degrees of freedomA cluster trial's information scales with the number of clusters; inference uses df ≈ (clusters − 2), not (participants − 2).

The design effect reduces the effective sample size (N/DEFF); it never inflates it. A trial with many participants in few clusters can be far weaker than its headcount suggests.

Does the conclusion survive?

—

If the naive analysis is significant but the cluster-adjusted one is not, the apparent effect was an artefact of pretending correlated participants were independent — a common error in cluster trials and in re-analyses that ignore the design.

How to read this · limitations. In a cluster-randomised trial, participants within a cluster are correlated, so the effective sample size is N divided by the design effect 1+(m̄−1)·ICC — analysing it as if individually randomised understates the standard error and can manufacture significance.

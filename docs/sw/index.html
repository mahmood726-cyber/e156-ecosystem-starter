E156 Ecosystem Starter

Sanidi kompyuta yako ya utafiti kama anavyofanya Mahmood Ahmad (kiongozi wa utafiti wa magonjwa ya moyo, Makerere & Ziauddin). Amri moja.

Rahisi zaidi — fungua kwenye kivinjari chako, usakinishaji kamili, hakuna terminal

Ikiwa haujawahi kutumia terminal, bofya kitufe hiki. Eneo la kazi la bure linalohifadhiwa na GitHub litafunguka likiwa na mfumo mzima uliosakinishwa kabla: sheria, kumbukumbu, Sentinel pre-push hook, Overmind verifier yenye ufunguo wa TruthCert HMAC, ProjectIndex seed, na Gemini CLI. Hakuna kitu kitakachosakinishwa kwenye kompyuta yako.

Fungua kwenye GitHub Codespaces →

Inachukua ~dakika 2-3 kujenga. Ni bora zaidi kwenye kompyuta ya mezani au mpakato (laptop) — Codespaces kwenye kivinjari cha simu inafanya kazi kitaalamu lakini terminal si thabiti.

Je, hauna akaunti ya GitHub bado? Soma hii kwanza.
  • Ikiwa una barua pepe ya chuo kikuu inayoishia na .ac.ug, .ac.za, .edu, n.k. — omba GitHub Education kwanza. Unapata saa 90 za bure za Codespaces kwa mwezi (badala ya 60), hakuna hatua ya uhakiki wa simu, na ufikiaji wa zana zingine za kulipia.
  • Ikiwa una barua pepe ya kibinafsi tu (Gmail/Yahoo) — jisajili kwenye github.com/signup. GitHub itaomba uhakiki wa simu kwa njia ya SMS; hii inaweza kufeli kimyakimya kwenye baadhi ya mitandao ya simu ya Afrika (Airtel, MTN). Ikiwa nambari ya siri haifiki, jaribu tena kutumia mtandao tofauti au tumia njia ya GitHub Education hapo juu.
  • Ikiwa simu yako ndio kifaa chako pekee — azima kompyuta ya rafiki yako kwa dakika 90 za kwanza za usanidi; baada ya hapo unaweza kufanya uandishi mwingi kutoka kwa simu yako kupitia hariri ya wavuti ya GitHub.
GitHub Codespace ni nini? (Toleo la aya 1)

Codespace ni kompyuta ya Linux inayofanya kazi kwenye wingu (cloud) la GitHub, ikiwa na VS Code (programu ya kuhariri nambari za kompyuta) iliyofunguliwa kwenye kivinjari chako cha wavuti. Hakuna kitu kinachowekwa kwenye laptop yako — kila kitu kinafanyika kwenye kompyuta ya wingu. Unaandika nambari kwenye kivinjari, unahifadhi kwenye GitHub, na bado zitakuwepo utakapofunga tabo hiyo. Ni bure kwa saa 60 kwa mwezi kwenye akaunti za kibinafsi (saa 90 kwenye akaunti za Masomo), kisha inasitishwa hadi mwezi unaofuata. Ni kama Google Docs, lakini kwa mazingira ya programu.

Tayari umesakinisha toleo la zamani? (kusasisha)

Ikiwa uliendesha kisakinishi hapo awali (k.m. v0.7.0 mnamo Machi), endesha tena mstari ule ule mmoja wa amri ulio juu ya ukurasa huu. Mchakato mpya utaleta toleo la mwisho lililowekewa alama na kufuta sheria na templeti zako za zamani. Faili ulizohariri mwenyewe (*.md.user) zinalindwa. Ili kuangalia toleo ulilosakinisha, endesha e156 version kwenye terminal yoyote ya Codespace.

Kidokezo cha i18n: kuweka kila kitu kwa Kiswahili ndani ya Codespace, ongeza E156_LANG=sw kwenye siri zako za mtumiaji wa Codespaces (Codespaces user secrets) katika github.com/settings/codespaces. Hii itatumika kwa kila codespace utakayofungua.

Muhimu — ukimaliza, simamisha codespace yako. Codespaces ambazo hazitumiki zinaendelea kutumia saa zako za kila mwezi hadi zitakapojisimamisha zenyewe (kawaida: dakika 30). Ili kusitisha mara moja: tembelea github.com/codespaces na ubofye ⋯ → Stop codespace, au endesha gh codespace stop kwenye terminal. Ukiwa na saa 60 za bure kwa mwezi, kuacha tabo wazi siku nzima kutamaliza salio lako ndani ya wiki moja.
Kumbuka: Ukurasa huu ni toleo lililofupishwa kwa Kiswahili. Toleo kamili la Kiingereza lina maelezo yote ya kina (mahitaji ya awali, utatuzi wa matatizo, mfano wa usalama, mifano ya miradi). Amri za usakinishaji hapa chini ni sawa katika lugha zote.

Usakinishaji — Windows

Fungua PowerShell (sio Command Prompt). Bonyeza Win+R, andika powershell, kisha bonyeza Enter. Bandika amri hii:

irm https://mahmood726-cyber.github.io/e156-ecosystem-starter/bootstrap.ps1 | iex

Usakinishaji — macOS / Linux / WSL

curl -fsSL https://mahmood726-cyber.github.io/e156-ecosystem-starter/e156-setup.sh | bash

Kuhakiki Usakinishaji

# Windows PowerShell
ls $env:USERPROFILE\.claude\rules

# macOS / Linux / WSL
ls ~/.claude/rules

Unapaswa kuona faili nne za .md: e156, lessons, advanced-stats, na rules.

Hatua inayofuata — zalisha kazi kwa kiwango kile kile cha ubora

Kusakinisha sheria ni nusu tu ya kazi. Nusu nyingine ni kuzitumia kwa kutumia mbinu ile ile: brainstorm → spec-lock → plan-lock → TDD → audit.

Soma STUDENT-WORKFLOW.md (kwa Kiingereza) kwa mwongozo kamili: mbinu ya hatua kwa hatua, mifano ya hazina za kukloni, na jinsi Sentinel + Overmind zinavyofanya kazi kama walinzi wa ubora.

Mahitaji ya Awali

  • Python 3.11+ kutoka python.org/downloads (weka alama ya tiki kwenye "Add python.exe to PATH").
  • R 4.5+ kutoka cran.r-project.org.
  • Node.js 20+ kutoka nodejs.org.
  • Wakala mmoja wa CLI wa AI: Gemini CLI (bure), Claude Code (ya kulipia), au OpenAI Codex.

Ikitokea matatizo

  • Tazama sehemu ya "What if something goes wrong" kwenye ukurasa wa Kiingereza.
  • Kisakinishi huandika kumbukumbu (log) kwenye %LOCALAPPDATA%\e156\logs\install-<ts>.log (Windows) au ~/.local/state/e156/logs/install-<ts>.log (Linux/WSL). Vifunguo vya API na siri hufichwa kiotomatiki kabla ya kushirikiwa.

# سلام Gemini! میں نے ابھی اس مشین پر e156-ecosystem-starter انسٹالر چلایا ہے۔

انسٹالر نے AI ایجنٹ کے قواعد، یادداشت کے ٹیمپلیٹس، اور (اختیاری طور پر) Sentinel/Overmind/ProjectIndex کو میرے ہوم ڈائریکٹری میں کاپی کر دیا ہے۔ آپ کا کام اب انسٹالیشن کا "دوسرا حصہ" ہے — وہ حصے جو اسکرپٹ نہیں بلکہ ایجنٹ سے ہوتے ہیں۔

براہ کرم یہ مراحل ترتیب سے کریں۔ رکیں اور مجھ سے پوچھیں اگر کوئی مرحلہ میرے ہوم ڈائریکٹری سے باہر کسی چیز کو چھوئے یا سسٹم PATH کو تبدیل کرے۔

**ہر مرحلے کے لیے قاعدہ:**
- "میں X چلاؤں گا" = آپ (ایجنٹ) اپنے شیل ٹول میں کمانڈ چلائیں۔
- "مجھے X چلانے کو کہیں" = میرے (صارف) لیے کمانڈ پرنٹ کریں تاکہ میں خود اپنی ٹرمنل میں چلاؤں۔
- "مجھ سے Y پوچھیں" = رکیں اور میرے جواب کا انتظار کریں۔

یہ اہم ہے کیونکہ میں مبتدی ہوں اور شاید نہ جانوں کہ کون سی کمانڈز خود ٹائپ کرنا محفوظ ہے۔

1. **آپ چلائیں** `cat ~/.claude/rules/rules.md | head -5` تاکہ یہ تصدیق ہو سکے کہ قواعد کا پیک انسٹال ہے۔ اگر فائل غائب یا خالی ہے، مجھے بتائیں کون سی فائل اور رکیں۔ اسے ٹھیک کرنے کی کوشش نہ کریں۔

2. **آپ چلائیں** یہ پیشگی شرائط کی جانچ: `python --version`، `Rscript --version`، `node --version`، `gh --version`، `git --version`۔ ہر اس چیز کے لیے جو غائب یا بہت پرانی ہے (Python <3.11، R <4.5، Node <20)، انسٹال کا صحیح URL پرنٹ کریں اور ایک سطر میں وضاحت کریں کہ یہ E156 کے کام کے لیے کیوں اہم ہے۔ غائب اشیاء کو خود انسٹال نہ کریں — فہرست دکھائیں اور مجھے فیصلہ کرنے دیں۔

3. **اگر `gh` انسٹال ہے لیکن authenticated نہیں** (آپ یہ `gh auth status` سے دیکھیں گے)، **مجھے بتائیں کہ میں چلاؤں** `gh auth login` اور میری تصدیق کا انتظار کریں اس سے پہلے کہ آگے بڑھیں۔

4. **Sentinel اور Overmind کا فوری ٹیسٹ (صرف اگر انسٹال ہیں)۔** اگر `sentinel` PATH پر ہے، `sentinel --version` چلائیں۔ اگر `overmind` PATH پر ہے، `overmind meta-verify` چلائیں اور فیصلہ بتائیں۔ دونوں ناکام = ماحول میں مسئلہ، دونوں کامیاب = جاری کرنے کے لیے تیار۔

5. **پہلا پراجیکٹ منتخب کریں۔** مجھ سے پوچھیں کہ `docs/index.html` کے 8 مثالی پراجیکٹس میں سے ("Forest-plot tool"، "PRISMA flow generator"، وغیرہ) کس سے شروع کرنا چاہتا ہوں۔ جب میں جواب دوں، **کسی بھی scaffolding سے پہلے**، پورٹ فولیو کا جائزہ چلائیں:
   ```
   python /workspaces/e156-ecosystem-starter/scripts/find-related-repos.py "<میرا موضوع>" --top 5
   ```
   (مقامی انسٹالیشنز کے لیے، اس راستے کو وہاں سے بدلیں جہاں آپ نے `e156-ecosystem-starter/` کلون کیا)۔ سب سے اوپر کے 5 نتائج پڑھیں — نام، README کا اقتباس، کوڈ grep نتائج۔ مجھے 3 سطروں میں بتائیں موجودہ repos سے کیا دوبارہ استعمال ہو سکتا ہے بمقابلہ کیا واقعی نیا ہے۔ پھر چلائیں
   `pip install git+https://github.com/mahmood726-cyber/e156-student-starter.git@main`
   اور `student new <slug>` تاکہ `~/code/<slug>/` کے تحت ڈھانچہ بن جائے۔ نئے repo کے `docs/<criterion>.md` میں سابقہ repos کا نام سے حوالہ دیں۔

6. **STUDENT-WORKFLOW.md پڑھیں**
   https://raw.githubusercontent.com/mahmood726-cyber/e156-ecosystem-starter/main/STUDENT-WORKFLOW.md
   پر تاکہ brainstorm → spec-lock → plan-lock → TDD → audit کا طریقہ سمجھ سکیں اس سے پہلے کہ نئے پراجیکٹ میں کوڈ کی ایک سطر بھی لکھیں۔

7. **یہاں رک جائیں اور مجھے بتائیں آگے کیا کرنا ہے۔** خود بخود پراجیکٹ نافذ نہ کریں۔ انسٹالر کا کام قواعد کو جگہ پر رکھنا تھا؛ پراجیکٹ کا کام ہمیں spec-locked طریقے سے مل کر کرنا ہے۔

پابندیاں:
- `~/.claude/`، `~/.gemini/`، `~/.codex/`، یا `~/code/` کے باہر کوئی فائل تبدیل نہ کریں۔
- `sudo` / admin کے طور پر کچھ نہ چلائیں۔
- اگر کچھ ناکام ہو، مجھے صحیح کمانڈ اور خرابی دکھائیں — حل اندازہ نہ لگائیں۔
